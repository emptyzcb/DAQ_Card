# 信息采集系统板卡硬件设计方案设计书

## 1. 项目目标

本项目设计一块面向机器人、工业巡检、移动载体和复杂电磁环境的数据采集前端板卡。板卡的核心职责不是执行 SLAM 计算，而是完成多路数字量、模拟量、惯性传感器、视觉同步信息、传统环境传感器以及外部里程计/雷达/视觉模块输出数据的统一采样、时间戳、缓存和打包输出。

本方案建议采用“实时采样打包板卡 + RS485 转光纤传输 + 后端处理单元”的系统架构：

- 采样板卡负责模拟量、数字量、IMU、环境传感器、视觉帧同步/触发、外部 SLAM 辅助数据接入、时间戳和协议打包。
- 板卡通过隔离 RS485 输出数据，再由 RS485 转光纤模块进行长距离、强抗干扰传输。
- 后端处理单元负责 SLAM 计算、视觉处理、数据融合、地图构建、轨迹估计和日志存储。

这样可以让前端板卡保持实时、稳定、低功耗和高可靠性，同时把视觉和 SLAM 的重计算放在远端处理平台。

## 2. 需求定义

### 2.1 采集对象

| 类型 | 建议配置 | 说明 |
| --- | --- | --- |
| 模拟输入 | 8 路差分或 16 路单端 | 支持 0-3.3V、0-5V、0-10V、±10V、4-20mA 可选量程 |
| 数字输入 | 8-16 路 | 支持 3.3V/5V/12V/24V 输入，建议隔离 |
| 数字输出 | 4-8 路 | 开漏、推挽或隔离 MOS 输出 |
| 计数/编码器 | 2-4 路 | 支持脉冲计数、频率测量、A/B 相编码器 |
| 陀螺仪/IMU | 1 路板载 + 1 路外接 | 板载 IMU 用于姿态，外接接口用于高精度工业 IMU |
| 视觉同步/状态 | 1-2 路触发/帧同步/状态输入 | 不建议直接传原始图像，可采集帧同步、曝光触发、帧号或外部视觉模块摘要数据 |
| 环境传感器 | 温湿度、气压、光照、气体、噪声等 | I2C/SPI/UART 扩展 |
| SLAM 辅助数据 | IMU、轮速、里程计、雷达状态、视觉帧同步、环境数据 | 统一时间戳后通过 RS485 光纤链路打包上传 |

### 2.2 性能目标

| 指标 | 建议目标 |
| --- | --- |
| 模拟采样分辨率 | 16 bit 优先，成本敏感可选 12 bit |
| 模拟总采样率 | 第一版 10-100 kSPS，后续按应用提升 |
| IMU 采样率 | 200 Hz - 2 kHz |
| 环境采样率 | 1-10 Hz |
| 视觉同步输入 | 30-120 Hz 帧同步/触发信号 |
| 时间戳精度 | MCU 内部 1 ms 起步，推荐提升到 10-100 us |
| 主数据输出 | 隔离 RS485，外接 RS485 转光纤 |
| 本地缓存 | SPI Flash / FRAM / SD 卡可选，用于链路异常时短时缓存 |

## 4. 处理器方案

### 4.1 实时采集 MCU

第一版可继续沿用 STM32F407。由于板卡只负责采样、时间戳和打包，不承担 SLAM 计算，STM32F407 可以满足原型验证。若后续模拟通道数、采样率、缓存和多协议接口明显增加，建议升级到 STM32F767、STM32H743 或同级 MCU。

MCU 主要职责：

- ADC 触发、DMA 采样、采样缓存管理。
- 数字输入边沿捕获、脉冲计数、编码器读取。
- IMU 和环境传感器采样。
- 统一时间戳。
- 数据帧编码、CRC 校验、底层输出。
- RS485 总线方向控制、发送队列、链路心跳和重传策略。
- 看门狗、故障诊断、状态上报。

### 4.2 后端 SLAM 处理单元

视觉和 SLAM 不在本采样板卡上执行。后端处理单元可以是工控机、嵌入式 Linux 主控、机器人主控或服务器。采样板卡只输出带时间戳的数据帧。

| 后端方案 | 适合场景 | 特点 |
| --- | --- | --- |
| 工控机 | 工业现场、算法验证 | 接口多，可靠性好 |
| 机器人主控 | 移动机器人 | 便于和运动控制、导航系统集成 |
| Jetson/RK3588 等边缘计算盒 | 视觉 SLAM、AI 识别 | 算力强，可接摄像头 |
| 上位机/服务器 | 实验室和集中处理 | 便于记录和回放 |

后端主要职责：

- 接收 RS485 光纤链路上传的数据帧。
- 校验 CRC、序号、时间戳和丢包情况。
- 融合 IMU、编码器、环境、视觉同步和外部传感器数据。
- 执行视觉处理、SLAM、地图构建、轨迹估计和日志存储。

## 5. 模拟采样硬件设计

### 5.1 输入类型

建议每路模拟输入通过跳线、电阻配置或模拟开关支持多种输入模式：

- 0-3.3V 低压信号。
- 0-5V 工业低压信号。
- 0-10V 工业模拟量。
- ±10V 双极性信号。
- 4-20mA 电流环。

### 5.2 模拟前端

每路模拟输入建议包含：

- TVS 管做浪涌/静电保护。
- 串联限流电阻。
- RC 低通滤波。
- 运放缓冲或差分放大。
- 量程转换电阻网络。
- 过压钳位到 ADC 安全范围。

对于工业环境，建议模拟输入与数字/通信侧做隔离，至少预留隔离版本。

### 5.3 ADC 选择建议

| ADC 方案 | 特点 | 适合版本 |
| --- | --- | --- |
| STM32 内部 ADC | 成本低，开发快 | 原型验证 |
| AD7606 系列 | 多通道同步采样，±10V 友好 | 工业通用采集 |
| ADS8688 系列 | 多量程输入，SPI 接口 | 中高精度采集 |
| ADS1115/ADS131M 系列 | 低速高精度 | 环境/慢变量采集 |

第一版如果目标是通用采样板卡，推荐外置 ADC。STM32 内部 ADC 可以用于辅助检测和低成本版本。

## 6. 数字量采集设计

### 6.1 数字输入

建议支持 3.3V、5V、12V、24V 输入，工业版本使用光耦或数字隔离器。

每路数字输入建议包含：

- 限流电阻。
- TVS/ESD 保护。
- RC 去抖滤波。
- 施密特触发整形。
- 光耦或数字隔离器。

功能模式：

- 普通电平读取。
- 上升沿/下降沿中断。
- 脉冲计数。
- 频率测量。
- 编码器 A/B 相输入。

### 6.2 数字输出

建议输出模式：

- 低压推挽输出，用于板内逻辑。
- 开漏输出，用于继电器、外部上拉、低速工业控制。
- 隔离 MOS 输出，用于工业负载。

每路输出建议增加：

- 过流保护。
- 反接保护。
- 续流二极管或 TVS。
- 输出状态回读。

## 7. IMU/陀螺仪设计

### 7.1 板载 IMU

板载 IMU 用于姿态估计、振动检测、运动状态判断。建议接口优先 SPI，I2C 作为低速备选。

设计要点：

- IMU 靠近板卡机械中心。
- 远离大电流电感、DC/DC、电机驱动和高速时钟。
- 单独 LDO 供电，模拟地回流干净。
- 预留中断引脚，用于数据就绪触发。
- 丝印标出 X/Y/Z 轴方向。

### 7.2 外接工业 IMU

预留外接接口：

- SPI/UART/RS485/CAN 可选。
- PPS 或同步输入。
- 独立供电和保护。
- 防呆连接器。

外接 IMU 的数据应与 ADC、视觉、编码器使用统一时间基准。

## 8. 视觉同步与外部视觉数据接入

本板卡不建议直接采集和传输原始图像。RS485 到光纤链路适合可靠传输低到中等速率的结构化数据，不适合承载高分辨率原始视频流。视觉图像应由后端处理单元或外部视觉模块采集，本板卡负责与视觉相关的同步和状态数据。

### 8.1 推荐接入内容

推荐采集以下视觉相关信号：

- 摄像头帧同步信号。
- 摄像头外触发信号。
- 曝光开始/结束信号。
- 帧号、曝光时间、增益、状态码等低速元数据。
- 外部视觉模块输出的目标点、特征摘要、里程计增量或状态数据。

如果确实需要接入图像，建议只接入低分辨率、低帧率、压缩后的摘要图像，并在协议层单独评估带宽。

### 8.2 同步设计

视觉和 IMU 融合时，时间同步很关键：

- MCU 输出同步脉冲给摄像头、雷达或外部视觉模块。
- MCU 捕获摄像头帧同步/曝光信号并记录硬件时间戳。
- 后端处理单元接收图像时，应使用相同时间基准或通过同步包校正偏移。
- MCU 上传 IMU、ADC、数字事件、环境数据时都带本地时间戳。
- 后端根据帧号、同步脉冲和采样时间戳进行多源数据对齐。

## 9. 环境信息采集设计

传统环境信息建议通过低速总线扩展：

| 传感器 | 推荐接口 | 采样率 |
| --- | --- | --- |
| 温湿度 | I2C | 1 Hz |
| 气压 | I2C/SPI | 1-10 Hz |
| 光照 | I2C/ADC | 1-10 Hz |
| CO2/VOC/气体 | I2C/UART/ADC | 0.1-1 Hz |
| 噪声 | ADC/I2S | 8 kHz 以上视需求 |
| 烟雾/粉尘 | UART/PWM/ADC | 1 Hz |

环境传感器建议放在板边缘，远离发热器件。温湿度传感器不要放在 DC/DC、处理器和功耗电阻旁边。

## 10. SLAM 数据采集与打包

SLAM 数据不是单一传感器数据，而是多源数据的同步结果。建议定义统一数据对象：

| 数据类型 | 来源 | 内容 |
| --- | --- | --- |
| IMU_SAMPLE | MCU | 加速度、角速度、温度、时间戳 |
| ADC_BLOCK | MCU | 通道号、采样率、样本块、时间戳 |
| DIGITAL_EVENT | MCU | 输入号、边沿、计数值、时间戳 |
| ENV_SAMPLE | MCU | 温湿度、气压、光照、气体等 |
| IMAGE_SYNC | MCU | 帧 ID、触发边沿、曝光信号、时间戳 |
| ODOMETRY | MCU/外部模块 | 轮速、编码器、里程计增量 |
| LIDAR_STATUS | MCU/外部模块 | 扫描同步、状态、低速摘要 |
| LINK_STATUS | MCU | RS485 光纤链路状态、丢包、重传统计 |

### 10.1 数据帧建议

底层 MCU 数据帧建议包含：

```text
帧头 | 协议版本 | 数据类型 | 序号 | 时间戳 | 长度 | Payload | CRC
```

后端处理后的 SLAM 高层数据不需要由采样板卡生成。采样板卡只上传原始/辅助数据。后端可以再生成：

- 调试阶段：JSON Lines + 二进制文件引用。
- 工程阶段：Protobuf、FlatBuffers 或自定义 TLV。
- ROS 生态：rosbag2 或自定义 bridge 节点。

### 10.2 时间同步建议

第一版：

- MCU 使用硬件定时器维护毫秒/微秒时间戳。
- 后端处理单元定期通过 RS485 光纤链路向 MCU 发送同步请求。
- MCU 回复当前 tick。
- 上位机根据同步包估计偏移。

升级版：

- 支持 PPS 输入。
- 支持外部 PPS 输入或同步脉冲输入。
- 摄像头、IMU、ADC、编码器事件使用硬件触发或硬件捕获时间戳。

## 11. 通信接口设计

主通信链路为隔离 RS485，外接 RS485 转光纤模块。其他接口用于调试、扩展和维护。

| 接口 | 用途 |
| --- | --- |
| USB-C | 调试、供电、数据导出 |
| RS485 主链路 | 打包数据输出，连接 RS485 转光纤模块 |
| RS485 扩展 | 外部工业传感器、外部 IMU 或从站设备 |
| CAN/CAN FD | 机器人、车辆、工业控制 |
| UART 调试口 | 底层日志 |
| SWD/JTAG | MCU 下载调试 |
| SPI Flash/FRAM/SD | 本地短时缓存和异常记录 |

### 11.1 RS485 转光纤设计要点

- RS485 收发器建议选工业级、宽温、带失效保护型号。
- MCU 侧 RS485 建议增加数字隔离器和隔离电源。
- A/B 线增加 TVS、共模电感、匹配电阻和偏置电阻。
- 支持半双工 RS485 时，MCU 必须控制 DE/RE 方向，并在协议中留出总线周转时间。
- 如果链路长期单向上传，可优先简化为主上传 + 低速命令下行。
- 光纤转换模块建议选择支持工业温度、宽压供电、状态指示和链路告警输出的型号。
- 协议层必须包含帧序号、CRC、链路心跳、统计计数和可选重传。
- 根据波特率评估带宽：RS485 常用 115200 bps、460800 bps、921600 bps、1-3 Mbps，实际距离和光纤转换模块能力要一起验证。

## 12. 电源设计

建议输入电源：

- 第一版：12V DC 输入。
- 工业版：9-36V 宽压输入。

板上电源轨：

| 电源 | 用途 |
| --- | --- |
| 5V | USB、部分传感器、RS485 光纤转换模块 |
| 3.3V | MCU、数字逻辑、部分传感器 |
| 模拟 3.3V/5V | ADC、运放、基准源 |
| 隔离电源 | 工业输入输出、RS485/CAN 隔离、外部传感器隔离 |

设计要点：

- DC/DC 负责主降压，LDO 负责模拟和传感器低噪声供电。
- ADC 使用独立基准源。
- 模拟地和数字地单点或受控连接。
- 高速接口、DC/DC、模拟前端分区布局。
- 输入端加入保险丝、TVS、反接保护和浪涌保护。

## 13. PCB 布局建议

- 模拟输入区域靠近接线端子，远离 DC/DC 和高速线。
- ADC 与模拟前端靠近，参考源走线短且干净。
- IMU 放在机械稳定区域，并标注坐标轴。
- RS485 A/B 差分线靠近连接器和保护器件，走线短、对称，远离模拟前端。
- 数字隔离器放在隔离边界上，隔离槽清晰。
- SWD、串口、测试点要方便调试。
- 所有外部接口加 ESD/TVS。
- 板上预留足够测试点：电源轨、ADC 输入、采样时钟、同步脉冲、UART。

## 14. 版本规划

### V0.1 原理验证板

- STM32F407 或现有开发板形态。
- 4 路模拟输入。
- 4 路数字输入。
- IMU660RC。
- USART/USB 调试输出。
- RS485 数据上传验证。
- Python 上位机采集。

### V0.2 通用采集板

- 8 路模拟输入。
- 8 路数字输入，4 路数字输出。
- 外置 16 bit ADC。
- 隔离 RS485 主链路，外接 RS485 转光纤。
- 环境传感器接口。
- SPI Flash/FRAM/SD 卡短时缓存。

### V0.3 SLAM 采样打包板

- 增加视觉帧同步/触发输入。
- 增加编码器、轮速、雷达同步或状态输入。
- 完善 MCU 与后端处理单元的时间同步。
- IMU + ADC + 数字事件 + 视觉同步 + 环境数据统一打包。
- 支持链路心跳、丢包统计、缓存补发。

### V1.0 工业化版本

- 宽压输入。
- 隔离数字输入输出。
- 隔离 RS485/CAN。
- 外壳、接插件、EMC、防反接、浪涌保护完善。
- 出厂校准流程。
- 固件升级和参数保存。

## 15. 固件配合要求

硬件设计需要和当前固件架构保持一致：

- `BSP/` 增加 ADC、DMA、采样定时器、外部通信驱动。
- `Services/` 增加采样缓存、协议编码、传感器融合前处理。
- `Application/` 增加工作模式、命令配置、状态机。
- `Docs/` 维护硬件接口表、CubeMX 配置表、协议文档。

当前已经建立的 `stream_protocol` 可以作为 MCU 底层数据帧基础，后续扩展 ADC、数字事件、环境数据和 IMU 数据类型。

## 16. 风险与注意事项

| 风险 | 影响 | 建议 |
| --- | --- | --- |
| RS485 带宽不足 | 无法传原始图像或高频大数据 | 只传结构化采样数据、视觉同步和摘要信息 |
| 模拟前端保护不足 | 工业现场易损坏 | 每路输入加保护和量程限制 |
| 时间同步粗糙 | SLAM 融合效果差 | 预留同步脉冲、PPS、硬件触发 |
| RS485 方向控制不当 | 半双工冲突、丢帧 | 使用 DMA 发送、发送完成中断和总线周转时间 |
| 电源噪声进入 ADC | 采样精度下降 | 模拟电源、参考源和布局分区 |
| 光纤转换模块选型不匹配 | 距离、速率或温度不满足 | 提前做链路样机测试 |

## 17. 第一阶段落地清单

第一阶段建议先做最小可验证闭环：

1. 确定模拟输入数量和量程。
2. 选择 ADC 方案。
3. CubeMX 配置 ADC + DMA + 采样定时器。
4. 固件增加 `bsp_adc_dma` 和采样缓存。
5. 使用二进制协议输出 ADC block + IMU sample。
6. 增加隔离 RS485 发送和 RS485 转光纤链路测试。
7. Python 上位机保存数据并画实时曲线。
8. 验证采样率、噪声、丢包率、链路延迟和时间戳稳定性。
9. 再接入视觉同步、编码器、雷达状态和外部 SLAM 辅助数据。

## 18. 推荐结论

本项目应设计成“SLAM 数据采样与打包前端板”。板卡不负责 SLAM 计算，也不建议传输原始视频流；它的价值在于把 IMU、模拟量、数字事件、编码器、环境量、视觉同步和外部模块状态做稳定采样、统一时间戳和可靠打包，再通过 RS485 转光纤链路送到后端处理单元。

第一版目标应是把 ADC/DMA/IMU/环境数据稳定采到，并通过 RS485 光纤链路可靠输出。第二阶段再扩展视觉同步、编码器、雷达状态和更完整的 SLAM 辅助数据包。
